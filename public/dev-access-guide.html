DegenDuel Dev Access Guide

How to access the development environment

Introduction

The DegenDuel development environment (dev.degenduel.me) is a restricted area for testing and development purposes. This guide explains how to access it.

Access Methods

There are several ways to access the development environment. Choose the method that works best for you.

Method 1: IP Address Authentication

The simplest method is IP address authentication. If your current IP address is in the allowed list, you'll automatically have access to the development environment.

Setup: Run the setup script on the server to add your IP address to the allowed list. 

node scripts/setup-dev-access.js

Pros: No login required, works automatically.

Cons: Only works from devices with the allowed IP address. If your IP changes, you'll need to update it.

Method 2: Wallet Authentication

If you're logged in with an authorized wallet address, you'll automatically have access to the development environment.

Setup: Run the setup script on the server to add your wallet address to the allowed list. 

node scripts/setup-dev-access.js

Pros: Works from any device as long as you're logged in.

Cons: Requires you to be logged in.

Method 3: Access Token

You can use a special access token to access the development environment from any device.

Setup: Run the setup script on the server to generate an access token. 

node scripts/setup-dev-access.js

Usage: There are two ways to use the access token:

  1. HTTP Header: Add the X-Dev-Access-Token header to your requests with the token as the value.
  2. Browser Script: Use the provided browser script to add the token to your requests. 
    // Copy the contents of public/dev-access.js
// Open the browser console on dev.degenduel.me
// Paste the script and press Enter
// Enter your token when prompted
    Download Browser Script

Pros: Works from any device, doesn't require you to be logged in.

Cons: Requires you to set up the token on each device.

Troubleshooting

If you're having trouble accessing the development environment, try the following:

  • Make sure you're using the correct access method.
  • Check that your IP address or wallet address is in the allowed list.
  • Make sure you're using the correct access token.
  • Clear your browser cache and cookies.
  • Try using a different browser.
  • Contact the system administrator for assistance.
